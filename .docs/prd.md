# PRD: Personal Research Feed (POC)

**Time box:** 60 minutes
**Audience:** Evaluator (Wilson)
**Deliverable:** Running Next.js app on localhost

---

## 1. Vision

A research feed that feels less like a tool and more like a mind expanding alongside the user's. It knows what they care about, notices patterns across unrelated items, and makes it trivial to take their thinking somewhere else (a memo, a meeting, a conversation).

The POC proves the **core loop** (topics → curated feed → save → synthesize) and gestures toward the vision with two signature moments: a **Daily Brief** on load and a **Signal** that surfaces cross-topic patterns.

## 2. User & Use Case

**Primary user:** A knowledge worker (PM, researcher, operator) who tracks 3 to 8 evolving topics and wants a low-effort way to stay current without drowning in feeds.

**Primary flow:**
1. Defines a topic ("LLM Infrastructure") with a short description of what they care about.
2. Opens the app, sees a curated feed for that topic plus a Daily Brief of cross-cutting highlights.
3. Saves the 2 to 3 items that actually matter, dismisses the rest.
4. Later, asks "what are the themes across my saved items?" or clicks **Create Memo** to hand off to another tool.

## 3. Features

### 3.1 Core Feature Set (Segment 1, must ship)

**Topics**
- Fields: `name`, `description`, `frequency` (Daily | Weekly | On Demand)
- CRUD via a Topics view
- Persistence: localStorage

**Feed**
- Per-topic feed of LLM-generated items
- Each item: `headline`, `summary` (2 to 3 sentences), `source` (simulated but plausible), `timestamp`
- Generated on demand via Claude Haiku 4.5 when user opens a topic or hits "Refresh"

**Actions**
- Mark as Read
- Save for Later
- Dismiss
- Actions persist and influence future generations (see Intelligence)

**Navigation**
- Three routes: `/feed`, `/saved`, `/topics`
- Plus a global chat drawer (toggle button in header)

**UI**
- Clean, minimal. Tailwind + a restrained color palette. No sidebar clutter.

### 3.2 Intelligence Layer (Segment 2, must ship)

**Simulated preference learning**
- When generating new feed items, the prompt receives:
  - Topic description
  - User's last 5 saved items (headlines + summaries) as positive signal
  - User's last 5 dismissed items as negative signal
- Model is instructed to tilt toward the saved patterns and avoid the dismissed ones

**Chat with content**
- Unified drawer, two modes:
  - **Topic mode:** "Ask about items in this topic"
  - **Global mode:** "Ask across everything"
- Backed by Claude Sonnet 4.6 with the relevant items passed as context
- Example prompts shown as starter chips: *"Main themes across my saved items this week"*, *"What did I miss on [Topic]"*

**Recency decay**
- Feed sort: newest first, with a boost for items whose topics the user engaged with recently
- Old items (>7 days) fade in the list unless saved

### 3.3 Portability (Segment 3, must ship)

**Create Memo**
- Button on the Saved view
- User selects items (checkbox) or uses all
- LLM generates a structured markdown memo:
  - Title
  - TL;DR (3 bullets)
  - Key themes
  - Pull quotes with source attribution
  - Suggested next actions
- Rendered in a modal with **Copy Markdown** and **Download .md** buttons

**Why memo, not CSV:** The use case in the brief is "writing a memo, preparing for a meeting." A CSV is data; a memo is thinking. The export should match the work the user is actually doing.

### 3.4 Ambient (Segment 4, should ship if time)

**Daily Brief on load**
- When the app opens and >12 hours have passed since last visit, a Daily Brief card appears at the top of `/feed`
- Generated by Claude Sonnet 4.6 with: all topics, any items saved since last visit, any unread items from preferred topics
- One short paragraph in human voice: *"Since yesterday, the biggest signal across your topics is X. You saved two items on Y. Worth your 5 minutes: [headline]."*

**Answers the brief's question:** *"What does this product do when you're not looking at it?"* It synthesizes ambient activity into one paragraph you read when you return.

Skipped: the social/sharing layer. Out of scope for a 1-hour POC.

### 3.5 Vision Moment (Segment 5, stretch)

**Signal**
- A small card on `/feed` that runs an LLM pass across **all saved items from the last 14 days** and surfaces one cross-topic pattern the user probably hasn't articulated yet
- Example output: *"You've saved 3 items about inference latency and 2 about eval orchestration. There's a theme forming around AI infra operational maturity, not model quality."*
- Regenerates on demand
- This is the product "noticing the shape of a problem across three unrelated articles"

## 4. Technical Decisions

| Layer | Choice | Reason |
|---|---|---|
| Framework | Next.js 15 App Router + TypeScript | Fastest scaffold with server routes + polished UI |
| Styling | Tailwind CSS | Ships with `create-next-app`, minimal overhead |
| UI primitives | Hand-rolled or shadcn (if fast) | Avoid component-library time sink |
| State | React state + localStorage | No DB, no auth, no multi-user |
| LLM client | `@anthropic-ai/sdk` via server route | Keeps API key off client |
| Feed-gen model | `claude-haiku-4-5-20251001` | Fast, cheap, good enough for synthetic items |
| Chat/memo model | `claude-sonnet-4-6` | Better reasoning for synthesis tasks |
| Deployment | Localhost via `npm run dev` | No deploy time in budget |

## 5. Non-Goals

Explicitly NOT building in this hour:
- Real web/RSS ingestion (synthetic items only)
- Multi-user, auth, accounts
- Mobile-specific layout (desktop-first, responsive-enough)
- Deploy pipeline
- Social/sharing network (Segment 4's network piece)
- Background jobs, cron, push notifications
- Real embeddings-based preference model (simulated via prompt context)
- Rich editor for memo output (markdown only)
- Tests (time doesn't allow, will note for post-hoc)

## 6. Success Criteria

The POC ships if, in a 3-minute demo, I can:

1. Create a topic from scratch
2. See a feed of LLM-generated items tailored to that topic
3. Save one, dismiss one, mark one as read
4. Ask the chat "what did I save about X" and get a useful answer
5. Click **Create Memo** and see a coherent markdown brief
6. See the Daily Brief paragraph on load
7. See the Signal card surface at least one non-obvious cross-topic theme

Stretch success: the UI feels quiet and considered, not demo-cluttered.

## 7. Risks

| Risk | Mitigation |
|---|---|
| API key not set in env | Check in first 60 seconds; block and ask if missing |
| LLM generations slow down feel | Haiku for feed, show streaming skeletons |
| Scope creep into Segment 4 social | Hard-skip; Daily Brief is the whole answer |
| UI polish eats remaining time | Set T+52 as polish cutoff |
| localStorage serialization bugs | Wrap reads in try/catch, reset to default |
